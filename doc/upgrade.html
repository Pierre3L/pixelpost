Upgrading from ANY version to the current version

These are merely guidelines, but are nonetheless strongly recommended to anyone upgraded their Pixelpost installation:

Do a backup of your files before you upload the new files to your webspace!
Do a backup of your database files

You also find information about backing up Pixelpost data at
http://wiki.pixelpost.org/index.php/Special:Search?search=backup&go=Go

Take care that you have an actual backup of all the files, which you modified yourself.

Especially take care that you have the actual version of the language files if you have modified them!

1) Upgrade from Version 1.1 or 1.2

This update is not supported anymore as the downloads are no longer available.

2) Upgrade from Version 1.3 to 1.6

Please note, that the file "pixelpost.php" which contains the database-access-informations is located now in the /includes/ directory! Please do not copy the file "pixelpost.php version 1.3" to this folder, as this will overwrite the version-information.

  1. First, it is highly recommended that you backup your installation.
  2. Open your current pixelpost.php, take the database-information from there and enter it into the \includes\pixelpost.php
  3. Be sure to enter the same "table-prefix" which you used for your 1.3-installation!
  4. for example:
    "$pixelpost_db_prefix = "pixelpost_"; // table prefix, leave as is unless you want to install multiple blogs on the same database"
  5. Copy all the files of the zip file into your directory.
  6. Browse to http://www.yourwebsite.com/admin
    It should prompt you to an upgrade page with a button: "upgrade". If not refresh your browser page. Upgrade and then login with your old username and password.
  7. Check the language-file and compare it with the new ones, or better update these as well.
  8. Note, that the CSS-definitions are changed, no CSS-definition contains "_" anymore in the name so check your templates and correct there, especially the calendar-definitions are changed! So please check the templates, which you use and change all underscores "_" to "-" otherwise your calendar will look crumbled as will other areas of your photoblog.
  9. Note that Pixelpost now provides separate CSS-files for the templates
  10. Note that there are far more TAGs now which you can use in your templates. Check out all the tags here.

3) Upgrade from Version 1.4 or 1.5 to 1.6

  1. If you modified something in the scripts please make a backup of your modifications (including modified language files or templates).
  2. Save your /includes/pixelpost.php file.
  3. Open the file /includes/pixelpost.php from the newly downloaded files and edit it:
    Enter your database-data into the following lines:
    $pixelpost_db_host = "localhost"; // database host, often "localhost"
    $pixelpost_db_user = "replace_me"; // database user
    $pixelpost_db_pass = "replace_me"; // database user password
    $pixelpost_db_pixelpost = "replace_me"; // database

    Take care that the tablename-prefix is correct, if your tablename-prefix differs from "pixelpost_", you must enter your old prefix here:
    $pixelpost_db_prefix = "pixelpost_"; // table prefix, leave as is unless you want to install multiple blogs on the same database
  4. Copy the new files from your local folder containing "Pixelpost 1.6" to your site using an FTP program.
  5. Go to your site and open /admin/install.php This will add one more table to the database
  6. After the upgrade is complete, login to the admin panel, and go to the addons page.
  7. Scroll down near the bottom and press the button labeled "Update EXIF".
  8. That's it! Your Pixelpost is upgraded and it should show the correct version in the admin panel

If you run into the following message box in the admin-section: "the layer does not exist (cropdiv). If you are using Netscape, please check the nesting of your tags" just empty the browser-cache and everything will be ok. Notice, when you are using Internet Explorer: it is not enough to empty the browser cache from within the browser, you must manually delete all files in the cache-directory of Internet Explorer. After doing so, everything should be hunky dory.
